# Bookly Production Readiness Plan

## Audit Summary

After reading every file in the project, here are the issues organized by priority.

---

## P0 ‚Äî Critical (must fix before launch)

### 1. Remove unused `axios` dependency
- **File**: `package.json` line 13
- `axios` is imported in `src/services/api.js` line 1 but never used ‚Äî all API calls use native `fetch`. Remove it.

### 2. Add React Error Boundary
- **New file**: `src/components/ErrorBoundary.jsx`
- No error boundary exists. If PopularBooks, TrendingBooks, or any section crashes, the entire app white-screens.
- Wrap each major section in `Home.jsx` with an error boundary that shows a graceful fallback.

### 3. Fix mock ratings leaking to users
- **Files**: `src/hooks/useBooks.js`, `src/components/home/PopularBooks.jsx`, `src/components/home/TrendingBooks.jsx`, `src/components/home/RecommendedBooks.jsx`, `src/components/home/RecentlyViewed.jsx`
- `generateMockRating()` is duplicated in 5 files and produces fake "4.5 (1,250 ratings)" that users see as real.
- **Fix**: Remove mock ratings entirely. Show stars only when real ratings exist (Gutendex has none ‚Üí don't show stars). Or label them clearly as "community ratings" if you plan to build a real rating system later.

### 4. Fix broken retry buttons
- **Files**: `src/components/home/PopularBooks.jsx` line 49, `TrendingBooks.jsx`, `RecommendedBooks.jsx`
- `onRetry={() => window.location.reload()}` ‚Äî full page reload is a bad UX. Instead, re-trigger the section's fetch function.

## P1 ‚Äî Important (should fix)

### 5. Add dynamic `<title>` and meta tags
- **File**: `index.html`
- Current: hardcoded `<title>Boockly</title>`, no meta description, no OG tags.
- Add: `<meta name="description">`, Open Graph tags (`og:title`, `og:description`, `og:image`), Twitter card tags.
- Use `react-helmet` or manual `document.title` updates in `Home.jsx`.

### 6. Add favicon and app icon
- **File**: `index.html`
- No favicon exists. Browser shows a blank tab icon.
- Create a simple book icon SVG favicon.

### 7. Add `manifest.json` for PWA basics
- **New file**: `public/manifest.json`
- Enables "Add to Home Screen" on mobile. Minimal: name, icons, theme_color, background_color, display: standalone.

- The Project Gutenberg `<a>` has `target="_blank"` but no `rel="noopener noreferrer"`.

### 9. Remove unused `mockBooks.js`
- **File**: `src/data/mockBooks.js`
- Only used by `Admin.jsx` as demo data. The admin panel is a stub ‚Äî either remove the file or document that it's demo-only.

### 10. Remove `console.log` from Admin
- **File**: `src/pages/Admin.jsx` line 90 ‚Äî `console.log('Upload complete:', data)`
- **File**: `src/components/AdminUploadForm.jsx` lines 140-143 ‚Äî logs FormData contents.

### 11. Add `aria-label` to Navbar
- **File**: `src/components/layout/Navbar.jsx` line 7

## P2 ‚Äî Nice to have (polish)

### 12. Code splitting with `React.lazy`
- **File**: `src/App.jsx`
- Admin page is rarely visited but loads eagerly. Wrap with `React.lazy` + `Suspense` to reduce initial bundle.

### 13. Add `<meta name="theme-color">` for mobile browser chrome
- **File**: `index.html`
- Sets the browser address bar color to match the brand.

### 14. Add scroll-to-top on route change
- **New component or hook**
- When navigating from `/admin` back to `/`, the page stays scrolled down.

### 15. Toast/notification system
- Currently no user feedback when actions succeed (e.g., rating saved). Add a lightweight toast component.

### 16. Limit pageCache size in useBooks
- **File**: `src/hooks/useBooks.js` line 18
- `pageCache` is a `Map` that grows unbounded. Add a max size (e.g., 10 entries) with LRU eviction.

### 17. Add `prefers-reduced-motion` support
- **File**: `src/index.css`
- Animations (`animate-fade-in-up`, `animate-pulse`) should be disabled for users who prefer reduced motion.
