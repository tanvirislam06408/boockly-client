# Bookly Performance Optimization Plan

## Performance Audit Results

### Bottleneck #1: API Request Waterfall (CRITICAL)
On initial page load, the app fires **~12 HTTP requests** to Gutendex simultaneously:
- 1× `fetchBooks()` — main grid
- 1× `fetchPopularBooks()`
- 1× `fetchTrendingBooks()`
- 2× `fetchRecommendedBooks()` (2 random topics)
- 1–8× `fetchBooksByIds()` — one request PER recently viewed book

**Impact**: Gutendex rate-limits, slow TTFB, waterfall delays. The `RecentlyViewed` section is the worst offender — it fires N individual requests instead of batching.

### Bottleneck #2: No Image Optimization (HIGH)
- Book covers are served at full Gutendex resolution (~300×450px JPEG)
- No `srcset` or responsive images
- No WebP/AVIF fallback
- 20+ cover images load on first paint (all sections combined)

### Bottleneck #3: Render-Blocking Google Fonts (HIGH)
- `index.html` line 37: Google Fonts CSS loaded with `display=swap` but still blocks first paint due to render-blocking `<link>` tag
- Two font families loaded (Inter + Playfair Display) — ~80KB combined

### Bottleneck #4: All Sections Eagerly Load (MEDIUM)
- PopularBooks, TrendingBooks, RecommendedBooks, RecentlyViewed all fire API calls on mount
- Below-the-fold sections should defer loading until near viewport

### Bottleneck #5: `mapBook` Overhead (LOW)
- `api.js` line 46–68: Every book goes through `mapBook()` which calls `getBestReadFormat()` and `pickDownloadUrl()` on every format object
- Called ~120+ times per page load (60 books × 2 sections)

### Bottleneck #6: Memory Leak in useBooks (LOW)
- `useBooks.js` line 123: `setTimeout(() => setNewBookIds(new Set()), 800)` is never cleaned up on unmount

## Implementation Plan

### Change 1: Batch RecentlyViewed API calls
**File**: `src/services/api.js`
- Replace `fetchBooksByIds()` (N individual requests) with a single batch approach
- Gutendex doesn't have a batch endpoint, but we can use `Promise.allSettled` with controlled concurrency (max 3 parallel) and cache results

### Change 2: Cache section API responses
**File**: `src/services/api.js`
- Add a simple in-memory cache for `fetchPopularBooks`, `fetchTrendingBooks`, `fetchRecommendedBooks`
- Cache key by topic/section, TTL of 5 minutes
- Prevents re-fetching on re-renders

### Change 3: Defer below-the-fold sections
**File**: New hook `src/hooks/useInView.js`
- IntersectionObserver-based hook that returns `true` when element enters viewport
- Wrap PopularBooks, TrendingBooks, RecommendedBooks, RecentlyViewed in this hook
- Only fire API calls when section is near viewport

### Change 4: Optimize image loading
**File**: `src/components/home/BookCard.jsx`
- Add explicit `width` and `height` attributes to prevent CLS
- Use Gutendex smaller image variant if available (they serve multiple sizes)

### Change 5: Self-host critical fonts
**File**: `index.html`, `src/index.css`
- Remove render-blocking Google Fonts `<link>` tag
- Use `@font-face` with `font-display: swap` and preload the critical font files
- Or keep Google Fonts but add `rel="preload"` for the CSS

### Change 6: Memoize BookSection
**File**: `src/components/home/BookSection.jsx`
- Wrap with `React.memo` to prevent unnecessary re-renders

### Change 7: Fix memory leak
**File**: `src/hooks/useBooks.js`
- Store setTimeout ID and clear it on unmount

## Expected Impact
- **Initial requests**: ~12 → ~3 (main grid + PopularBooks + deferred sections load later)
- **Time to Interactive**: ~40% faster (fewer blocking requests)
- **CLS**: Near zero (explicit image dimensions)
- **Font loading**: Non-blocking with swap
